# Skill Extractor

一个"元技能"，能够从对话中自动识别和提炼可复用的技能模式，并将其保存为标准的 SKILL.md 文件。

## 功能特点

- **自动识别**: 检测对话中重复出现的任务模式
- **手动触发**: 用户主动请求技能提炼
- **智能提取**: 分析工具使用模式、领域知识和工作流程
- **标准输出**: 生成符合项目规范的 SKILL.md 文件
- **确认机制**: 用红色标记显示建议，等待用户确认

## 使用方法

### 手动触发

```
提炼一个技能：从这段对话中提取技能模式
```

### 自动识别

当 AI 检测到以下模式时，会主动建议：

1. 重复出现的任务（3次以上）
2. 明确的工具使用序列
3. 特定领域的专业知识
4. 复杂的多步骤流程

## 技能提炼流程

```
检测模式 → 提取要素 → 生成 SKILL.md → 用户确认 → 创建文件
```

## 输出格式

生成的技能包含：

- 技能名称（英文，kebab-case）
- 功能描述
- 激活关键词（中英文）
- 工具列表
- 使用模式
- 分步指令
- 错误处理
- 示例场景

## 示例

### 输入

```
User: "我经常需要分析股票数据，获取技术指标，生成图表"
```

### 输出建议

```
🔴🔴🔴 **[技能提炼建议]** 🔴🔴🔴

检测到可复用技能模式：

技能名称: stock-analysis
描述: 使用 AkShare 进行股票技术分析
激活关键词: 股票分析, stock analysis, technical indicators

是否确认创建？
```

### 确认后生成

完整的 `stock-analysis/SKILL.md` 文件，包含所有必要章节。

## 最佳实践

1. **等待模式稳定**: 确保模式重复出现后再提炼
2. **验证关键词**: 确保激活词足够具体
3. **测试技能**: 创建后测试技能是否可用
4. **持续迭代**: 根据使用反馈改进技能
